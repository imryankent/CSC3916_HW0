Request Headers
  User-Agent: PostmanRuntime/7.26.10
    Describes browser/tool making the call, Postman in this case.
  Accept: */*
    Describes which type of content the client will accept. In this case, it is all MIME types
  Postman-Token: 41b898bf-8e6f-498a-928b-66fe74571ce1
    Custom token generated by Postman
  Host: www.googleapis.com
    Which URL does the request go to?
  Accept-Encoding: gzip, deflate, br
    Which types of data encoiding should I accept?
  Connection: keep-alive
    Shall I keep the connection open or close it?
  
Response Headers
  Content-Type: application/json; charset=UTF-8
    Content type of the returning payload. In this case jason with the UTF-8 character set
  Vary: Origin
  Vary: X-Origin
  Vary: Referer
    These 3 headers determine if a cached copy can by used by a future request instead of fetching from the network
  Content-Encoding: gzip
    Type of encoding in the response
  Date: Mon, 01 Feb 2021 01:47:56 GMT
    Pretty self explanatory...
  Server: ESF
    Type of server where the response came from
  Cache-Control: private
    Can I cache the response?
  X-XSS-Protection: 0
    Cross site script protections, determines whether or not thie API can be called from.
  X-Frame-Options: SAMEORIGIN
    Security protocol determining how a webpade is rendered
  X-Content-Type-Options: nosniff
     Security protocol determining opting out off MIME type sniffing
